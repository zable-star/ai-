# 前后端集成测试指南

## 测试步骤

### 1. 启动后端服务器

```bash
cd backend

# 安装依赖（首次）
npm install

# 配置数据库（首次）
.\scripts\setup-db.ps1   # Windows
# 或
bash scripts/setup-db.sh  # Linux/Mac

# 启动服务器
npm run dev
```

后端应该在 http://localhost:3000 启动

### 2. 启动前端服务器

```bash
# 在项目根目录
python -m http.server 8000
```

前端应该在 http://localhost:8000 可访问

### 3. 打开测试页面

浏览器访问: http://localhost:8000/frontend/test-integration.html

## 测试清单

### ✅ 基础连接测试
1. 打开测试页面
2. 点击"检查后端连接"
3. 应该看到绿色成功消息

### ✅ 用户认证测试
1. **注册新用户**
   - 输入用户名（例如：testuser）
   - 输入密码（至少6个字符）
   - 点击"注册"
   - 应该看到成功消息
   - 认证状态变为"已登录"

2. **退出登录**
   - 点击"退出登录"
   - 认证状态变为"未登录"

3. **登录测试**
   - 输入刚注册的用户名和密码
   - 点击"登录"
   - 应该看到成功消息

### ✅ 模型配置测试
1. **创建配置**
   - 确保已登录
   - 点击"创建新配置"
   - 填写表单（可以用假的 API Key 测试）
   - 点击"保存配置"
   - 应该看到成功消息

2. **加载配置列表**
   - 点击"加载配置列表"
   - 应该看到刚创建的配置

### ✅ AI 聊天测试
1. 确保已登录且有模型配置
2. 输入测试消息
3. 点击"发送测试"
4. **注意**: 这个会实际调用 AI API，需要有效的 API Key

### ✅ 绘图保存测试
1. 确保已登录
2. 输入绘图名称
3. 点击"保存测试绘图"
4. 应该看到成功消息
5. 点击"加载绘图列表"
6. 应该看到刚保存的绘图

## 常见问题

### 问题 1: 后端连接失败
**症状**: "后端连接失败" 错误

**解决方案**:
1. 检查后端是否启动: `http://localhost:3000/health`
2. 检查端口 3000 是否被占用
3. 检查防火墙设置
4. 查看后端终端的错误日志

### 问题 2: 数据库连接错误
**症状**: 后端启动失败，提示数据库连接错误

**解决方案**:
1. 确保 PostgreSQL 已安装并运行
2. 检查 `.env` 文件中的数据库配置
3. 运行数据库配置脚本: `.\scripts\setup-db.ps1`

### 问题 3: CORS 错误
**症状**: 浏览器控制台显示 CORS 错误

**解决方案**:
1. 检查后端 `.env` 中的 `CORS_ORIGIN` 设置
2. 应该设置为: `CORS_ORIGIN=http://localhost:8000`
3. 重启后端服务器

### 问题 4: 401 Unauthorized
**症状**: 登录后的请求返回 401

**解决方案**:
1. 检查 JWT token 是否正确存储
2. 打开浏览器开发者工具 → Application → Local Storage
3. 查看是否有 `authToken`
4. 尝试重新登录

### 问题 5: 注册失败 - 用户名已存在
**解决方案**: 使用不同的用户名，或清空数据库重新开始

## 调试技巧

### 查看网络请求
1. 打开浏览器开发者工具 (F12)
2. 切换到 Network 标签页
3. 执行操作
4. 查看请求和响应详情

### 查看后端日志
后端终端会显示所有 API 请求和错误信息

### 查看浏览器控制台
Console 标签页会显示 JavaScript 错误和 API 响应
