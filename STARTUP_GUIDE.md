# 🚀 语音绘图应用启动指南

## 快速启动检查清单

在使用语音绘图功能前，请确认以下几点：

### ✅ 1. 浏览器要求

- **推荐浏览器**：Google Chrome 或 Microsoft Edge（最新版）
- **最低版本**：Chrome 25+ / Edge 79+
- **不支持**：Firefox、Safari（部分功能受限）

### ✅ 2. 网络要求（重要！）

**Web Speech API 需要连接到 Google 服务器**

- ✅ **必须**：能够访问 Google.com
- ✅ **推荐**：使用稳定的 VPN/代理工具
- ❌ **不支持**：纯内网环境（无法访问 Google）

**验证网络连通性：**
```
1. 在浏览器新标签打开: https://www.google.com
2. 如果能正常访问，说明网络没问题
3. 如果不能访问，请先解决网络连接问题
```

### ✅ 3. 麦克风权限

- 浏览器会弹出麦克风权限请求
- 必须点击"允许"才能使用语音功能
- 如果误点了"拒绝"，请按以下步骤修复：
  1. 点击地址栏左侧的锁图标 🔒
  2. 找到"麦克风"选项
  3. 选择"允许"
  4. 刷新页面

### ✅ 4. HTTPS 或 localhost

- 生产环境必须使用 HTTPS
- 开发环境可以使用 localhost 或 127.0.0.1
- HTTP 协议不支持语音识别

## 🔧 常见问题及解决方案

### 问题 1: "语音识别服务连接失败"

**现象**：页面显示"语音识别服务连接失败"或"network 错误"

**原因**：无法连接到 Google 语音服务

**解决方案**：

1. **检查代理/VPN**
   ```
   - 确认 VPN/代理软件已开启
   - 测试访问 google.com
   - 确保浏览器流量走代理
   ```

2. **验证网络连通性**
   ```bash
   # 打开浏览器控制台（F12），执行：
   fetch('https://www.google.com').then(() => console.log('✅ 能访问')).catch(() => console.log('❌ 不能访问'))
   ```

3. **刷新页面**
   - 开启代理后，按 Ctrl+Shift+R 强制刷新页面

4. **使用测试工具**
   - 打开 `simple-voice-test.html` 进行详细诊断
   - 按步骤逐项检查

### 问题 2: "麦克风权限被拒绝"

**现象**：页面提示需要麦克风权限

**解决方案**：

1. 点击地址栏左侧的 🔒 图标
2. 找到"麦克风"或"Microphone"
3. 从下拉菜单选择"允许"
4. 刷新页面（F5）

### 问题 3: "没有检测到语音"

**现象**：提示"no-speech"错误

**解决方案**：

1. **检查麦克风硬件**
   - 确认麦克风已连接
   - 在系统设置中测试麦克风
   - Windows：设置 → 系统 → 声音 → 输入
   - macOS：系统偏好设置 → 声音 → 输入

2. **检查浏览器设置**
   - 确认使用了正确的麦克风设备
   - Chrome：设置 → 隐私和安全 → 网站设置 → 麦克风

3. **说话技巧**
   - 靠近麦克风
   - 说话清晰、音量适中
   - 避免环境噪音

### 问题 4: "浏览器不支持语音识别"

**现象**：提示浏览器不支持 Web Speech API

**解决方案**：

1. 更换浏览器为 Chrome 或 Edge
2. 确保浏览器版本是最新的
3. 检查浏览器版本：
   - Chrome：chrome://version
   - Edge：edge://version

### 问题 5: 语音识别效果不好

**现象**：识别不准确或经常出错

**优化建议**：

1. **改善环境**
   - 安静的环境
   - 减少背景噪音
   - 使用质量好的麦克风

2. **说话方式**
   - 吐字清晰
   - 适中语速
   - 标准普通话

3. **使用语义模型**
   - 在"配置"界面接入 DeepSeek 或通义千问
   - 可以更好地理解复杂指令

## 🎯 快速测试步骤

### 方法 1: 使用简单测试页面（推荐）

1. 打开 `frontend/simple-voice-test.html`
2. 点击"1. 检查浏览器支持"
3. 点击"2. 请求麦克风权限"并允许
4. 点击"3. 开始语音识别"
5. 说话测试

这个页面会给出详细的错误信息和解决方案。

### 方法 2: 浏览器控制台测试

```javascript
// 打开浏览器控制台（F12），粘贴并执行：

// 1. 检查浏览器支持
const Recognition = window.SpeechRecognition || window.webkitSpeechRecognition;
console.log('支持语音识别:', !!Recognition);

// 2. 测试语音识别
if (Recognition) {
  const recognition = new Recognition();
  recognition.lang = 'zh-CN';
  recognition.onstart = () => console.log('✅ 语音识别已启动');
  recognition.onerror = (e) => console.error('❌ 错误:', e.error);
  recognition.onresult = (e) => console.log('✅ 识别到:', e.results[0][0].transcript);
  recognition.start();
  console.log('正在监听，请说话...');
}
```

### 方法 3: 使用主应用的语音自检

1. 打开主应用
2. 点击"语音自检"按钮
3. 听到语音播报说明功能正常
4. 尝试说"小绘，小绘"唤醒

## 📝 启动流程

### 正常启动流程

1. **页面加载** → 显示"待启动"
2. **请求麦克风权限** → 用户点击"允许"
3. **启动语音识别** → 显示"监听中"
4. **等待唤醒词** → 提示"说'小绘，小绘'唤醒我"
5. **用户说"小绘，小绘"** → 系统回应"在呢，你想画什么"
6. **说绘图指令** → 例如"画一个红色的圆"
7. **执行绘图** → 语音播报"已绘制中间红色圆形"

### 如果启动失败

1. **检查浏览器** → 是否为 Chrome/Edge
2. **检查网络** → 能否访问 Google
3. **检查权限** → 麦克风是否允许
4. **使用测试工具** → simple-voice-test.html
5. **查看日志** → 浏览器控制台（F12）

## 🌐 网络配置建议

### 如果在国内使用

**强烈建议使用稳定的代理工具**，因为：

1. Web Speech API 必须连接到 Google 服务器
2. 没有代理则无法使用语音识别功能
3. 代理不稳定会导致识别中断

### 代理配置检查

1. **系统代理**
   - 确认系统代理已设置
   - 测试能否访问 Google

2. **浏览器代理**
   - Chrome/Edge 通常使用系统代理
   - 确认浏览器流量走代理

3. **验证方法**
   ```
   打开 google.com
   如果能正常访问 → 代理正常
   如果不能访问 → 检查代理设置
   ```

### 企业/学校网络

如果在企业或学校网络：

1. 检查防火墙设置
2. 确认是否屏蔽了 Google 服务
3. 可能需要联系网络管理员
