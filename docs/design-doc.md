# 设计文档：AI 语音绘图工具

## 1. 题目选择

本项目选择题目二：AI 语音绘图工具。

目标是让用户不依赖鼠标或键盘，仅通过语音完成基础绘图创作。浏览器首次获取麦克风权限可能需要一次点击，这是浏览器安全限制；进入绘图流程后，绘图、修改、撤销、清空和导出都通过语音完成。

## 2. 计划支持的指令能力

| 能力 | 计划内容 |
| --- | --- |
| 基础图形 | 圆形、矩形、三角形、直线、星形、点 |
| 颜色控制 | 红、蓝、绿、黄、紫、黑、白、灰、橙、粉 |
| 位置控制 | 中间、左上、右上、左下、右下、上方、下方、左边、右边 |
| 线宽控制 | 通过“线宽五”“粗一点”“细一点”调整 |
| 画布控制 | 清空、撤销、重做、保存图片 |
| 复杂指令 | 用“然后、再、并且”等词拆分多步命令 |
| 容错表达 | 支持部分近义词，例如“圆/圆形/圈”“矩形/方形” |

## 4. 未完成能力与原因

| 未完成能力 | 原因 |
| --- | --- |
| 完全自由语义绘图 | 需要云端大模型或图像生成模型，成本和不确定性更高，不适合作为第一版核心路径 |
| 像素级编辑 | 需要更复杂的对象选择、坐标推理和编辑模型，超出 72 小时 MVP 范围 |
| 对上一笔进行自然语言编辑 | 已预留历史栈，但“把刚才的圆变大一点”需要对象引用解析，后续可补 |
| 多语言语音识别 | 当前优先中文命令，英文仅做少量颜色兼容 |
| 服务端账号系统 | 当前部署在 GitHub Pages，没有安全后端；本地账号用于演示用户流程，生产版需要托管认证服务 |
| 服务端 API 代理 | 目前由用户在浏览器输入自己的 API Key；生产版应通过后端代理隐藏密钥 |
| 素材库 | 暂定为下一步扩展。当前优先保证语音绘图、模型接入、账号隔离和导出链路稳定；素材库会引入图片版权、加载性能和画布合成复杂度 |

## 5. 指令理解与容错策略

- 将一句话按“然后、接着、再、并且”等连接词拆分为多个子命令。
- 每个子命令先识别控制类动作，如清空、撤销、重做、保存。
- 再识别颜色、线宽、位置和图形。
- 对常见同义表达做词典映射。
- 无法识别时给出可恢复反馈，不改变画布。
- 当本地规则无法识别且用户启用大模型时，调用 OpenAI-compatible API，将自然语言转换为结构化动作 JSON。
- 大模型配置与本地账号绑定。未登录或退出后不会加载 API Key；重新登录后才恢复该账号保存的模型配置。
- 配置界面支持保存多个模型配置，用户可以按服务商或用途灵活切换。
- 模型返回结果会经过类型、坐标、颜色和线宽校验，避免无效输出直接执行。

## 6. 语音到绘图延迟策略

- 使用浏览器 Web Speech API 获取最终识别文本。
- 最终文本一旦返回，立即在本地解析并绘制。
- 不在主流程调用云端模型，避免网络等待。
- Canvas 渲染只重放动作历史，数据量小，响应稳定。

## 7. 复杂指令拆解与执行

示例：

```text
画一个红色圆形，然后在右下角画蓝色矩形，再画一条绿色斜线
```

执行过程：

1. 拆分为三个子命令。
2. 分别解析为结构化动作。
3. 按顺序写入历史栈。
4. 重绘画布。

## 8. 成本控制

虽然题目二没有强制要求端云成本说明，但本项目仍把成本控制作为设计目标：

- MVP 不使用云端大模型，避免按 token 或音频计费。
- 不上传画布内容，减少存储和带宽成本。
- 只在浏览器内保存绘图历史，避免服务器资源消耗。
- 后续如果接入云端语义解析，可采用“本地规则优先，失败后再调用模型”的混合策略。
- 当前已采用混合策略：能由本地规则理解的命令不调用模型；只有未知复杂命令才进入模型规划。
- API Key 由用户自己填写并保存在当前登录账号的本地数据中，不写入仓库；退出登录后运行态不再持有该配置。
- 通过多个配置档切换不同服务商，便于在 DeepSeek、OpenAI、通义千问等模型之间按效果和成本选择。

## 9. 后续扩展

- 引入对象选择：例如“把左边的圆移动到右边”。
- 引入云端命令规划器：将复杂自然语言转为操作 JSON。
- 支持模板化作品：流程图、海报草图、教学板书。
- 支持导出操作历史，便于复现绘图过程。
- 评估素材库：支持快速加载图片、模板和背景，但需要先解决素材版权、移动端加载性能、图片与语音操作的组合方式。
- 将本地账号升级为后端账号系统，并把模型 API 调用迁移到后端代理。
