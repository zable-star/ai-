## 🐛 修复的问题

### 1. 修复停止后无法唤醒 ✅

**问题描述**: 
- 5秒无声音自动停止后，说"小绘，小绘"无法重新唤醒
- 识别已停止，无法监听唤醒词

**根本原因**:
- 自动停止后只是stop了识别，但没有重新启动
- 导致无法监听后续的唤醒词

**修复方案**:
- 自动停止后，延迟1秒自动重新启动识别
- 进入唤醒词模式，持续监听"小绘，小绘"
- 用户可以随时唤醒，无需手动操作

**修复代码**:
```javascript
// 延迟1秒后自动重新启动识别（唤醒词模式）
setTimeout(() => {
  if (!appState.listening) {
    startRecognition();
    heardText.textContent = "等待唤醒...";
  }
}, 1000);
```

### 2. 修复谐音识别问题 ✅

**问题描述**:
- 识别成"小惠"后不执行唤醒流程
- 被当作普通指令处理，导致"未识别"错误

**根本原因**:
- 谐音列表不完整，缺少"小惠"和"小灰"
- 非唤醒词时会显示"听到: xxx"，干扰用户

**修复方案**:
- 扩展谐音识别列表：
  - ✅ 小绘
  - ✅ 小会
  - ✅ 小慧
  - ✅ 小惠（新增）
  - ✅ 小灰（新增）
- 非唤醒词时不显示"听到: xxx"，只显示"等待唤醒..."

**修复代码**:
```javascript
const isWakeWord = transcript.includes("小绘") ||
                  transcript.includes("小会") ||
                  transcript.includes("小慧") ||
                  transcript.includes("小惠") ||  // 新增
                  transcript.includes("小灰");   // 新增

if (isWakeWord) {
  // 执行唤醒逻辑
} else {
  // 不是唤醒词，安静等待（不提示听到了什么）
  heardText.textContent = `等待唤醒...`;
}
```

## 💡 改进说明

### 为什么添加这些谐音？

中文语音识别的特点：
- "绘" 的发音 hui，容易识别成同音字
- 常见谐音：惠、会、慧、灰
- 覆盖这些谐音可以大幅提高识别成功率

### 为什么延迟1秒重启？

用户体验考虑：
- 给"5秒无声音"提示留出显示时间
- 避免立即重启让用户看不到停止提示
- 1秒是最佳平衡点（不会太长，也不会太短）

### 为什么改进语音提示？

心理学原理：
- "我在，请说" → 简单确认
- "在呢，你想画什么" → 友好 + 引导
- 后者让用户更明确知道接下来该做什么
