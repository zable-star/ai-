# 项目改进计划

## 用户反馈的问题

### 1. 语音反馈太频繁 ⚠️
**问题**: 每一步都有语音回复，太吵
**建议**: 只保留文字提示，或只在关键操作时语音提示

**优先级**: 🔴 高（影响用户体验）

---

### 2. 语音质量差 ⚠️
**问题**: 浏览器默认语音太机器人
**建议**: 使用更自然的语音合成

**优先级**: 🟡 中（可以改善但不影响核心功能）

---

### 3. AI 理解能力不足 ⚠️
**问题**: 语音命令理解不够准确
**建议**: 优化 AI prompt，提高识别率

**优先级**: 🔴 高（影响核心功能）

---

### 4. 素材库太少 ⚠️
**问题**: 可绘制的图案太简单
**建议**: 扩展复合对象库，增加细节

**优先级**: 🟡 中（功能性改进）

---

## 改进方案

### 问题 1: 禁用或减少语音反馈

#### 方案 A: 完全禁用语音（最简单）
**修改位置**: `frontend/app.js`
**修改方式**: 注释掉所有 `speak()` 调用

**优点**:
- 立即生效
- 完全静音
- 不影响其他功能

**实现时间**: 2 分钟

---

#### 方案 B: 添加语音开关（推荐）
**修改位置**: `frontend/index.html` + `frontend/app.js`
**实现内容**:
- 在设置中添加"启用语音反馈"开关
- 用户可以自己选择开关
- 默认关闭

**优点**:
- 用户可控
- 灵活性高
- 保留语音功能

**实现时间**: 15 分钟

---

#### 方案 C: 选择性语音（平衡方案）
**修改位置**: `frontend/app.js`
**策略**:
- 只在重要操作时语音提示（保存成功、登录成功）
- 绘图操作只显示文字
- 错误时才语音提示

**优点**:
- 平衡体验
- 保留关键反馈

**实现时间**: 10 分钟

---

### 问题 2: 改进语音质量

#### 方案 A: 使用浏览器高质量语音（免费）
**实现方式**:
```javascript
const utterance = new SpeechSynthesisUtterance(text);
// 选择更好的语音
const voices = window.speechSynthesis.getVoices();
const betterVoice = voices.find(v => 
  v.lang === 'zh-CN' && v.name.includes('Female')
);
if (betterVoice) utterance.voice = betterVoice;
```

**优点**:
- 免费
- 无需额外配置
- 立即可用

**缺点**:
- 质量仍然有限
- 依赖浏览器和系统

**实现时间**: 5 分钟

---

#### 方案 B: 集成 Azure Speech Service（推荐）
**服务**: Microsoft Azure Text-to-Speech
**特点**:
- 非常自然的中文语音
- 多种音色选择
- 免费额度：每月 500 万字符

**实现步骤**:
1. 注册 Azure 账号
2. 创建 Speech Service
3. 获取 API Key
4. 集成到前端

**优点**:
- 质量最好
- 非常自然
- 多种音色

**缺点**:
- 需要网络
- 需要配置

**实现时间**: 30 分钟

---

#### 方案 C: 使用 Google Cloud TTS
**服务**: Google Cloud Text-to-Speech
**特点**:
- WaveNet 模型，质量好
- 免费额度：每月 100 万字符

**实现时间**: 30 分钟

---
