# 语音语言问题说明

## 问题：为什么有时候是粤语回答？

### 原因分析

浏览器的语音合成（Text-to-Speech）会加载系统中安装的所有中文语音包，包括：

- **zh-CN** - 普通话（简体中文）
- **zh-HK** - 粤语（香港广东话）
- **zh-TW** - 台湾国语（繁体中文）

之前的代码会匹配所有 `zh-*` 开头的语音，没有区分普通话和粤语，所以有时候会随机选到粤语语音。

### 已修复

现在已经修改了语音选择逻辑，通过评分机制优先选择普通话：

**评分规则：**
- `zh-CN`（普通话）：+100 分 ⭐⭐⭐
- `zh-TW`（台湾国语）：+60 分 ⭐⭐
- `zh-HK`（粤语）：+10 分 ⭐
- 包含"cantonese"、"粤语"、"广东话"：-200 分 ❌

这样可以确保优先选择普通话语音。

## 如何查看当前使用的语音

### 方法 1：查看控制台日志

1. 按 F12 打开浏览器开发者工具
2. 点击"语音自检"按钮
3. 在控制台中会看到：
   ```
   === 语音自检 ===
   检测到 X 个中文语音:
   ✅ [1] Google 普通话（中国大陆） (zh-CN) - 得分: 225
      [2] Microsoft Huihui (zh-CN) - 得分: 35
      [3] Google 粤語（香港） (zh-HK) - 得分: -190
   
   已选择: Google 普通话（中国大陆） (zh-CN)
   ```

4. ✅ 表示当前选中的语音

### 方法 2：语音自检提示

点击"语音自检"后，页面会显示：
```
语音自检通过，当前音色：Google 普通话（中国大陆）
```

### 方法 3：每次播放时的日志

每次语音播报时，控制台会输出：
```
🔊 使用语音: Google 普通话（中国大陆） (zh-CN)
```

## 如果还是遇到粤语

### 检查系统语音包

**Windows 系统：**
1. 打开：设置 → 时间和语言 → 语音
2. 查看已安装的语音
3. 如果有多个中文语音，可以卸载粤语语音包

**macOS 系统：**
1. 打开：系统偏好设置 → 辅助功能 → 朗读内容
2. 点击"系统语音"
3. 查看并管理语音包

### 手动指定语音

如果需要手动指定特定的语音，可以修改代码：

```javascript
// 在 frontend/app.js 的 speak() 函数中
// 找到 const voiceCandidates = getPreferredChineseVoices(voices);

// 改为手动指定：
const voiceCandidates = voices.filter(v => 
  v.name.includes('Google') && 
  v.lang === 'zh-CN'
);
```


## 故障排除

### 1. 语音自检显示粤语

**检查步骤：**
1. 按 F12 打开控制台
2. 点击"语音自检"
3. 查看日志中的语音列表和得分
4. 截图发给开发者

### 2. 没有可用的中文语音

**解决方案：**
1. 安装 Google Chrome 浏览器（自带 Google 语音）
2. 或者在系统中安装中文语音包
3. Windows：设置 → 语音 → 添加语音
4. macOS：系统偏好设置 → 辅助功能 → 下载语音

### 3. 语音自检通过，但实际使用时还是粤语

这不应该发生。如果遇到这种情况：
1. 刷新页面
2. 清除浏览器缓存
3. 重新进行语音自检
4. 查看控制台的语音选择日志

## 技术说明

### 语音选择流程

```
1. 页面加载 → 等待语音列表加载完成
2. 过滤中文语音 → isChineseVoice()
3. 评分排序 → scoreChineseVoice()
4. 选择最高分 → voiceCandidates[0]
5. 播放语音 → speak()
```

### 为什么要排除粤语

虽然粤语也是中文，但：
1. 大部分用户期望听到普通话
2. 应用界面是简体中文
3. 语音指令是普通话
4. 粤语和普通话发音差异很大

如果您确实需要粤语语音，可以修改评分规则。
