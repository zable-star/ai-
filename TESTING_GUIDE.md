# 详细测试步骤

## 🚀 第一步：启动后端

### 1.1 安装 Node.js 依赖

```bash
cd backend
npm install
```

预期输出：应该看到依赖包安装成功

### 1.2 配置数据库（首次运行）

**Windows 用户：**
```powershell
# 在 PowerShell 中运行
.\scripts\setup-db.ps1
```

**Linux/Mac 用户：**
```bash
bash scripts/setup-db.sh
```

**如果没有 PostgreSQL，请先安装：**
- Windows: https://www.postgresql.org/download/windows/
- Mac: `brew install postgresql`
- Ubuntu: `sudo apt-get install postgresql`

**临时测试方案（无数据库）：**
如果暂时无法安装 PostgreSQL，可以先跳过后端，直接测试前端现有功能。

### 1.3 配置环境变量

```bash
# 复制示例配置
cp .env.example .env

# 编辑 .env 文件
notepad .env  # Windows
# 或
nano .env     # Linux/Mac
```

**必须修改的配置：**
```env
DB_PASSWORD=你的PostgreSQL密码
JWT_SECRET=随机字符串（可以运行下面的命令生成）
```

**生成 JWT 密钥：**
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 1.4 启动后端服务器

```bash
npm run dev
```

**成功标志：**
```
🚀 Server running on port 3000
📝 Environment: development
🔒 CORS origin: http://localhost:8000
```

**保持这个终端窗口运行！**

## 🌐 第二步：启动前端

### 2.1 打开新终端

在项目根目录（不是 backend 目录）：

```bash
# 确保在项目根目录
cd ..  # 如果还在 backend 目录

# 启动 HTTP 服务器
python -m http.server 8000
```

**如果没有 Python，可以使用：**
- Node.js: `npx http-server -p 8000`
- VS Code: 安装 Live Server 扩展

**成功标志：**
```
Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...
```

**保持这个终端窗口运行！**

## 🧪 第三步：测试集成

### 3.1 打开测试页面

在浏览器中访问：
```
http://localhost:8000/frontend/test-integration.html
```

### 3.2 测试后端连接

1. 点击页面顶部的 **"检查后端连接"** 按钮
2. **预期结果**：应该看到绿色消息 "✅ 后端连接成功！时间戳: ..."

**如果失败：**
- 检查后端是否在运行（终端 1）
- 检查地址是否为 `http://localhost:3000`
- 打开浏览器开发者工具（F12）查看错误

### 3.3 测试用户注册

1. 在 **"1️⃣ 用户注册"** 部分
2. 输入用户名（例如：`testuser`）
3. 输入密码（至少6个字符，例如：`test123456`）
4. 点击 **"注册"** 按钮

**预期结果：**
- 绿色消息：✅ 注册成功！用户: testuser
- 顶部认证状态变为 "已登录"
- 显示当前用户名

**如果失败：**
- 查看错误消息
- 可能是用户名已存在，换一个用户名
- 检查数据库是否正确配置

### 3.4 测试用户登录

1. 点击 **"退出登录"** 按钮（认证状态变为"未登录"）
2. 在 **"2️⃣ 用户登录"** 部分
3. 输入刚才注册的用户名和密码
4. 点击 **"登录"** 按钮

**预期结果：**
- 绿色消息：✅ 登录成功！用户: testuser
- 认证状态变为 "已登录"

### 3.5 测试模型配置

1. 确保已登录
2. 在 **"3️⃣ 模型配置管理"** 部分
3. 点击 **"创建新配置"** 按钮
4. 填写表单：
   - 配置名称：`测试配置`
   - 服务商：选择 `OpenAI`
   - API 端点：`https://api.openai.com/v1/chat/completions`
   - 模型名称：`gpt-4o-mini`
   - API Key：随便填（例如：`sk-test123`）
5. 点击 **"保存配置"**

**预期结果：**
- 绿色消息：✅ 配置创建成功！ID: xxx

6. 点击 **"加载配置列表"** 按钮

**预期结果：**
- 显示刚创建的配置信息

### 3.6 测试绘图保存

1. 确保已登录
2. 在 **"5️⃣ 绘图保存测试"** 部分
3. 输入绘图名称：`我的第一幅画`
4. 点击 **"保存测试绘图"** 按钮

**预期结果：**
- 绿色消息：✅ 绘图保存成功！ID: xxx

5. 点击 **"加载绘图列表"** 按钮

**预期结果：**
- 显示保存的绘图信息
- 包含名称、动作数、创建时间

## 🎨 第四步：测试主应用（部分功能）

### 4.1 打开主应用

在浏览器中访问：
```
http://localhost:8000/frontend/
```

### 4.2 测试注册和登录

1. 点击左侧的 **"账号"** 标签
2. 注册一个新账号或使用之前的账号登录
3. **预期结果**：顶部显示 "已登录：用户名"

### 4.3 测试语音绘图（基础功能）

1. 点击左侧的 **"绘图"** 标签
2. 点击 **"🎤 开始"** 按钮
3. 允许浏览器使用麦克风
4. 说出命令：**"画一个红色圆形"**
5. **预期结果**：画布上出现红色圆形

**更多命令测试：**
- "画一个蓝色矩形"
- "画一个黄色三角形"
- "清空画布"
- "撤销"

### 4.4 测试 AI 模型调用（需要真实 API Key）

⚠️ **注意**：这个测试需要有效的 OpenAI API Key，会产生费用！

1. 点击 **"配置"** 标签
2. 登录后配置一个真实的 API Key
3. 启用模型
4. 返回 **"绘图"** 标签
5. 说出复杂命令：**"帮我画一个夕阳，有一棵树"**
6. **预期结果**：AI 规划多个绘图动作并执行

## ⚠️ 常见问题

### 问题 1：后端连接失败

**错误**: "❌ 后端连接失败"

**解决方案**:
```bash
# 1. 检查后端是否运行
# 在浏览器访问: http://localhost:3000/health
# 应该返回: {"status":"ok","timestamp":"..."}

# 2. 检查端口是否被占用
netstat -ano | findstr :3000  # Windows
lsof -i :3000                  # Mac/Linux

# 3. 重启后端
# Ctrl+C 停止后端
npm run dev  # 重新启动
```

### 问题 2：数据库连接错误

**错误**: 后端启动时显示数据库连接错误

**解决方案**:
```bash
# 1. 检查 PostgreSQL 是否运行
# Windows: 任务管理器 → 服务 → postgresql
# Mac: brew services list
# Linux: sudo systemctl status postgresql

# 2. 检查 .env 配置
# DB_PASSWORD 是否正确
# DB_NAME 是否存在（默认 voice_drawing）

# 3. 重新运行数据库配置脚本
.\scripts\setup-db.ps1  # Windows
```

### 问题 3：CORS 错误

**错误**: 浏览器控制台显示 CORS 错误

**解决方案**:
```bash
# 检查后端 .env 文件
CORS_ORIGIN=http://localhost:8000

# 确保前端运行在 8000 端口
# 重启后端使配置生效
```

### 问题 4：401 Unauthorized

**错误**: 登录后的请求返回 401

**解决方案**:
1. 重新登录
2. 检查浏览器 LocalStorage 中是否有 `authToken`
3. Token 可能已过期（默认 7 天）

### 问题 5：无法安装 PostgreSQL

**临时解决方案**:
```
暂时跳过后端测试，先测试前端现有功能：
1. 直接访问 http://localhost:8000/frontend/
2. 测试语音绘图（不需要登录）
3. 测试本地命令解析
4. 测试画布功能（清空、撤销、重做、导出）
```
